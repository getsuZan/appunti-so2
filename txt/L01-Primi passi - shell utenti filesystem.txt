Il prompt
• La bash scrive un prompt ed attende che l'utente
scriva un “comando”
• il prompt tipico e così costituito:
nomeutente@nomemacchina:~cammino$
• dove cammino è il percorso dalla cartella (directory o
folder) home alla cartella attuale

• quindi, se si è semplicemente nella home, c'è solo
la tilde ~
• se la cartella corrente non si trova nel sottoalbero
radicato nella home, allora cammino è il percorso
assoluto

Il comando
• Ogni comando verrà nel seguito indicato come segue
• comando [opzioni] argomenti_obbligatori

• Ci dev'essere almeno un argomento (ma ce ne puo essere anche più d'uno) se
{argomento}
• cp [-r] [-i] [-a] [-u] {filesorgenti} file_destinazione

• Ci possono essere 0, 1 o più argomenti opzionali se [opzione...]
• ps [opzioni] [pid...]

• Argomenti vanno separati da carattere indicato se
[carattere opzione...]
• chmod mode[, mode...] file_name

Il comando (cont.)
• Opzioni tipicamente composte da
• -carattere (vecchia versione)
• --parola (nuova versione)

• Ad esempio sono equivalenti per il comando cp
• -i
• --interactive

-r
--recursive

• Le opzioni possono avere un argomento
• -k1, -k 1, --key=1

• Le opzioni senza argomento sono raggruppabili
• -b -r -c sono equivalente a -brc

• Esistono anche le opzioni in stile BSD, senza dash ‘-’
• tar xfz nomefile.tgz

Gli utenti
• Durante l’installazione di Linux è necessario specificare (almeno) un utente
(utente principale)
• Alcune versioni creano un utente automaticamente ad es.
• Utente osboxes
• Password osboxes.org

• root è il superutente: ha tutti i privilegi (amministratore di sistema)
• Non tutti gli utenti possono «entrare» nel sistema (to log in)
• root non può fare login da un terminale qualsiasi (di solito solo dalla console) ma
un utente può acquisire i diritti di root mediante il comando su e sudo

• Ogni utente appartiene almeno ad un gruppo
• automaticamente creato con lo stesso nome dell’utente principale

• Esistono molti gruppi definiti per scopi «amministrativi»
• groups [nome_utente]

sudo
• Nelle distribuzioni della famiglia Debian l’utente
principale è un sudoer, appartiene al gruppo
predefinito sudo
• Gli utenti del gruppo sudo possono eseguire
comandi da super utente (root) usando il
comando sudo
• sudo comando
• guardare il man per la sinossi completa

• sudo è un comando che prende in ingresso un
altro comando

Creazione di altri utenti
• adduser nuovo_utente
• Di default l’utente creato non appartiene al gruppo sudo

• Esercizio [da fare sul vostro pc]:

• con utente principale (sudoer) eseguire il comando apt-get update
• con utente principale eseguire il comando sudo apt-get update

• Per aggiungere un utente ad un gruppo
• adduser nuovo_utente gruppo
