umask [mode]
• Imposta la maschera dei file a mode; ovvero i diritti di accesso al file o alle
directory nel momento della loro creazione
• Diritti accesso directory = 0777 AND NOT(umask)
• L’operazione AND è bit a bit
• 0666 per i file
• Gli attributi speciali non sono impostati (notare il primo 0 ottale = 000)

• Per i file però il diritto di esecuzione non viene impostato (e quindi le
opzioni speciali non hanno effetto) [notare 666=110110110]
• Esercizio 1: modificare l'umask in modo che, sia che venga creata una
directory sia un file, i permessi siano 664. Dopodiché, modificare
nuovamente in modo che il permesso per una nuova directory sia 775 e
per un file sia 664.

ln [-s] sorgente [destinazione]
• Soft/symbolic e hard link
• Un link lin1 ad un file f1 è un nuovo file lin1 che punta ad f1
senza contenere in esso il contenuto di f1 (permette di risparmiare
spazio)
• Default hard link; -s soft link
• Negli hard link non c'è il concetto di puntatore e puntato

dd [opzioni]
• Per convertire o copiare files in modo «sofisticato»
• Le opzioni sono una sequenza variabile=valore
• Le variabili più importanti sono:
• bs dimensione di un singolo blocco in lettura/scrittura
• count numero di blocchi da copiare
• convert, in questo caso, il valore specifica una conversione, per esempio di
codifica (da minuscolo a maiuscolo e viceversa, più svariate altre cose; vedere il man)
• if file di input (se non dato, legge da tastiera (standard input))
• of file di output (se non dato, scrive su schermo (standard output))

• Oltre che per le conversioni si usa per copiare file speciali che non possono
essere copiati con cp

mkfs [-t type fsoptions] device
• Crea un filesystem su device; a volte detto anche formattazione
perché prepara i file a memorizzare file secondo un dato formato, ad
es. ext4
• I tipi type li abbiamo visti la scorsa lezione – vedi anche il man
• fsoption può essere ro oppure rw
• device: device file in /dev oppure file regolare, ad es. creato
con dd
• Se è un dispositivo, non deve essere montato (/dev/hda, /dev/hda2,
/dev/sdb1 )

• Se nella vostra distro non è disponibile mkfs usare /sbin/mkfs
