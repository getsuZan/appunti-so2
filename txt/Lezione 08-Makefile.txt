Make e Makefile
• Meccanismo per automatizzare la compilazione di file sorgenti
• Definisce un semplice linguaggio per descrivere le relazioni tra
• sorgenti, file oggetto ed eseguibili

• Motivazioni
• I tempi di compilazione possono essere molto lunghi per programmi grandi
(provate a compilare Google Chromium …). Conviene ricompilare solamente
se un sorgente è stato aggiornato
• Più agevole rispetto alla compilazione manuale

• Può essere utilizzato con qualsiasi linguaggio che abbia un
compilatore eseguibile da riga di comando

Make e makefile
• make attua regole della forma
Target: Prerequisites
<TAB>Recipe
<TAB>Recipe
...
<TAB>Recipe

• Queste regole sono contenute nel makefile
• Il carattere <TAB> che precede ogni Recipe è obbligatorio
• Target è tipicamente un file (ad es., un eseguibile o file oggetto)
• Prerequisites è una lista di file o altri target
• Recipe è un comando da eseguire (è una parte della «ricetta» per
raggiungere il target)
zuliani@di.uniroma1.it

Esempio
• La recipe associata ad un target viene eseguita quando:
• non esiste (nella directory in cui make è invocato) un file di nome target
*e* vi sono dei prerequisites
• Il file di target esiste ma uno o più prerequisiti sono stati aggiornati dopo la
sua generazione

• Esempio di makefile
merge_sorted_lists: merge_sorted_lists.c
gcc -Wall -Wextra -O3 merge_sorted_lists.c -o merge_sorted_lists

• Il target è l'eseguibile merge_sorted_lists
• La lista dei prerequisiti contiene il file merge_sorted_lists.c
• Il recipe è il comando per la compilazione

Phony target (azione)
• Un target senza prerequisiti è chiamato azione o target phony (finto,
fasullo)
clean:
rm -f *.o merge_sorted_lists

• Bisogna dichiarare esplicitamente che il target è phony
.PHONY clean
• altrimenti se viene creato un file di nome clean nella directory in cui si
esegue il make, l’azione sopra definita non verrà mai eseguita
• infatti, un target senza prerequisiti si considera sempre aggiornato se esiste
un file con quel nome

Regole e Variabili implicite
• make possiede un insieme di regole implicite che coprono casi
comuni (ad es., la generazione di un eseguibile a partire da un
sorgente C)
• Permettono di scrivere regole formate dal solo target o da target e
prerequisiti, senza una recipe esplicita
• Esempio:
• main: main.c list.h list.c
• crea un eseguibile main a partire da prerequisite

• Se il target dipende da un solo prerequisito con il suo stesso nome (a
meno dell'estensione), il prerequisito si può omettere
• make cerca un file con il nome del target e sceglie il recipe implicito da
utilizzare in base alla sua estensione
• es: merge_sorted_lists:

Regole e Variabili implicite (cont.)
• Le regole implicite usano variabili implicite
• Essendo ridefinibili dall'utente permettono di "controllare" l'esecuzione delle
recipe implicite
• Principali variabili (modificabili dall’utente)
• CC : compilatore
• CFLAGS : flag per la fase di compilazione
• LDFLAGS : flag per il linker (-L)
• LDLIBS : flag per il linker: aggiunge librerie (-l<libname>)

• La recipe implicita eseguita per la compilazione di un eseguibile x a partire da un
singolo sorgente x.c è:
• $(CC) $(CFLAGS) $(LDFLAGS) x.c $(LDLIBS) -o x

• Quindi ad esempio il target main: main.c list.h list.c eseguirà la
regola implicita
•

$(CC) $(CFLAGS) $(LDFLAGS) main.c list.h list.c $(LDLIBS) -o main
